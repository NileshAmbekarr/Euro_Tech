# Changelog

All notable changes to the EuroCore Backend are documented here.

---

## [Unreleased] — 2026-03-06

### Deployment: Render + Aiven MySQL

---

### `src/main/resources/application-prod.properties`

#### Added
- Full production config for **Aiven MySQL** (`DATABASE_URL`, `DATABASE_USERNAME`, `DATABASE_PASSWORD` via env vars)
- HikariCP connection pool settings optimised for free-tier limits (max 5 connections)
- `spring.flyway.enabled=false` — disables Flyway so Hibernate `ddl-auto=update` can auto-create tables on first boot
- CORS config via `${CORS_ORIGINS}` env var (defaults to `http://localhost:4200`)
- Actuator health probes (`liveness`, `readiness`) with `show-details=always` for easier diagnostics

#### Fixed
- **Removed `server.servlet.context-path=/api`** — this was causing a double `/api` prefix because controllers already declare `@RequestMapping("/api/v1/...")`, making every API route return a 500 with `"No static resource v1/..."`.

---

### `src/main/java/com/bootcamp/api/config/CorsConfig.java`

#### Fixed
- **Was hardcoding `addAllowedOrigin("http://localhost:4200")`** — completely ignoring the `CORS_ORIGINS` environment variable set in Render.
- **Rewritten** to read origins from `${cors.allowed-origins}` via `@Value`, supporting comma-separated values (e.g. `https://eurocore.in,https://www.eurocore.in`).
- Added `setAllowCredentials(true)` and explicit `OPTIONS` method allowance for preflight requests.

---

### `src/main/java/com/bootcamp/api/config/SecurityConfig.java`

#### Fixed
- **Missing `.cors(Customizer.withDefaults())`** in the active `SecurityFilterChain`.
- Without this, Spring Security was intercepting `OPTIONS` preflight requests **before** the `CorsFilter` could respond, causing CORS errors even when `CorsConfig` was correctly configured.
- Added `import org.springframework.security.config.Customizer`.
- Cleaned up the large commented-out security chain block.

---

### `backend/render.yaml`

#### Fixed
- Changed `region: oregon` → `region: singapore` to match the Render dashboard selection (best latency for India).
- Removed Render-managed DB stanza — now wired to **Aiven MySQL** as external DB.
- `DATABASE_PASSWORD` set as `sync: false` (must be entered manually in Render dashboard).

---

### Known Issues / Notes

| Issue | Status | Notes |
|---|---|---|
| `mail: DOWN` in health check | ⚠️ Open | Render free tier blocks outbound SMTP port 587. Gmail emails will not send. Use SendGrid or Resend as SMTP relay, or disable mail health indicator. |
| `JWT_SECRET` | ✅ Auto-generated | Render `generateValue: true` in `render.yaml` handles this. |
| First-boot table creation | ✅ Works | `ddl-auto=update` creates all tables in Aiven `defaultdb` on first successful startup. |

| `JWT_SECRET` | Auto-generated by Render |
